# 法學論文大綱審查工具

本專案提供一套法學碩士論文大綱審查框架，協助使用者檢視章節結構是否符合學術寫作邏輯，並引導逐步修正。

---

## 專案內容

| 檔案 | 用途 |
|------|------|
| `SKILL_legal_thesis_outline.md` | Claude 專案 SKILL 版，供 Claude.ai 專案功能使用 |
| `legal_thesis_outline_prompt.md` | 通用 Prompt 版，可貼入任何 AI 對話框使用 |
| `README.md` | 本說明文件 |

---

## 兩個版本的差異與使用方法

### 版本一：SKILL 版（`SKILL_legal_thesis_outline.md`）

**適用對象：** 使用 Claude.ai 並建立了「專案（Project）」的使用者

**使用方式：**
1. 進入 Claude.ai，開啟或建立一個「專案」
2. 在專案設定中找到「Custom Instructions」或「Project Knowledge」
3. 將 `SKILL_legal_thesis_outline.md` 的內容上傳或貼入
4. 之後在該專案內的任何對話，只要提到「大綱」、「章節」、「論文架構」等關鍵字，Claude 會自動觸發審查流程

**特點：**
- 自動觸發，不需要每次手動貼入
- 與專案內其他自訂規則（如引用格式、語言偏好）並存運作
- 僅限 Claude.ai 平台，其他 AI 工具不支援此格式

---

### 版本二：Prompt 版（`legal_thesis_outline_prompt.md`）

**適用對象：** 使用任何 AI 對話工具的使用者，包括 ChatGPT、Gemini、Claude、Copilot 等

**使用方式：**
1. 開啟 `legal_thesis_outline_prompt.md`，複製全部內容
2. 在 AI 對話框中貼入，送出
3. AI 會回應確認已載入，並詢問你的論文題目與大綱
4. 提供你的論文資料後，即開始逐步審查

**特點：**
- 跨平台通用，不限 AI 工具
- 每次新對話都需要重新貼入（對話結束後不會保留）
- 建議貼入後的第一句話說明你的需求，例如：「我有一份論文大綱需要審查」

---

## 審查功能說明

本工具可協助檢查以下問題：

**章層（第一層）**
- 章的順序是否符合「緒論→理論→制度→比較法→檢討→結論」的標準架構
- 章名是否空泛，涵蓋範圍是否與論文研究範圍相符
- 比較法章節是否獨立成章且位置正確

**節層（第二層）**
- 各節是否服務該章主題
- 內容是否錯置於錯誤章節
- 每章是否有小結作為收尾

**章節對話關係**
- 第二章理論是否在第五章被引用為評價標準
- 第四章比較法結論是否在第五章被承接
- 第五章是否有過渡性前言節，統整比較法發現後再進入修法建議

## 使用範例

以下為測試用的模擬大綱，可用來驗證工具是否正常運作：

**論文題目：**《個人資料保護法之修正方向研究——以生成式AI蒐集個資為核心》

**大綱：**

- 第一章 緒論
  - 第一節 研究動機與目的
  - 第二節 研究範圍與限制
  - 第三節 論文架構
- 第二章 個人資料保護之理論基礎
  - 第一節 資訊自決權之憲法依據
  - 第二節 個人資料保護之立法目的
  - 第三節 生成式AI與個資保護之交叉問題
- 第三章 我國個人資料保護法之現行規範
  - 第一節 蒐集、處理與利用之要件
  - 第二節 當事人同意原則
  - 第三節 現行規範之缺漏
- 第四章 外國立法例比較
  - 第一節 歐盟GDPR之規範架構
  - 第二節 美國模式之分散立法
  - 第三節 比較法小結
- 第五章 我國制度之檢討與修正建議
  - 第一節 以歐盟模式為借鏡之可行性
  - 第二節 修法建議
  - 第三節 小結
- 第六章 結論

**這份大綱包含以下可供審查的問題：**
1. 第二章第三節混入具體爭議，不屬於純理論層次
2. 第三章章名空泛，涵蓋整部個資法而非聚焦於生成式AI情境
3. 第五章第一節直接跳入歐盟模式評估，缺少統整比較法發現的過渡節
