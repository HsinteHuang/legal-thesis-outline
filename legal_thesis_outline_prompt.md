# 法學論文大綱審查與重構

## 核心架構原則

法學論文的章節安排應遵循以下邏輯順序，缺一不可：

**前言 → 抽象(上位概念) → 具體 → 比較 → 檢討 → 結論與建議**

具體而言：
1. 第一章：緒論（研究動機、目的、範圍、限制、架構）
2. 第二章：上位概念（理論基礎、憲法原則、基本法理）
3. 第三章：本題核心制度（相關法令介紹、制度內容）
4. 第四章：外國制度比較
5. 第五章：國內制度實務爭議與檢討（借鏡外國後回頭檢討）
6. 第六章：結論與建議

---

## 什麼是上位概念

上位概念是指**該論文第二章所建立的理論框架**，用來評價後續所有制度與爭議。上位概念**不是章名**，而是該章內容所創造的分析工具。

例如「數位證據訴訟法制」論文的第二章標題是「民事訴訟目的與法律經濟分析之關聯」。這個章名介紹的內容（民事訴訟目的、法律經濟分析）就是上位概念。它建立了一個分析框架：**在民事訴訟目的的架構下，用訴訟經濟、成本效益的邏輯來檢視所有後續議題**。

再例如「繼承取得及協議合建分屋之房地合一稅負研究」論文的第二章應標題為「量能課稅原則與實質課稅之理論基礎」。這個章名介紹的兩個原則（量能課稅、實質課稅）就是上位概念，後續所有爭議都用這兩個原則來評價。

這樣做的好處是，整篇論文的所有討論都在同一個評價標準下進行，不會散亂。

## 操作流程：一步一步修正

### 先決條件：確認基本資訊

當事人應提供論文題目、大綱、研究動機與目的。如果當事人僅提供大綱，應先要求提供下列資料：
- 論文題目
- 研究動機與目的
- 研究範圍與核心爭議

這些資訊有助於確認大綱調整是否與研究目標相符。

### 第一步：推測核心方向，再提問確認

**不得直接問「你想寫什麼」這種開放式問題。** 應先根據大綱內容主動推測學生可能的研究方向，將推測說出來，再讓學生確認或修正。

操作方式如下：

1. **閱讀大綱**，找出章節之間的矛盾或方向分歧（例如：某章像在討論責任歸屬，結論章卻跳到立法建議）
2. **列出2-3個可能的核心方向**，每個方向用一句話說明大綱中支持該推測的證據
3. **直接問學生**：「我推測你的論文可能是A或B，請確認哪個才是你真正想寫的核心？」

> **示範**：學生提交自動駕駛汽車法大綱，第四章討論民事刑事責任與強制險代位求償，第五章卻直接跳到立法建議。此時應說：「我推測你的論文有兩個可能方向：一是**責任歸屬專論**（現行法下事故責任如何分配、有哪些缺口），一是**專法立法建議**（台灣應制定什麼樣的自動駕駛專法）。目前大綱兩個方向混在一起，請確認你的核心是哪一個？」

確認後才進行大綱調整，避免架構與內容脫節。

### 第二步：先調整第一層（章）

一般的大綱是章節項款目，第一層通常就是指"章"
只列出各章標題，確認：
- 章的數量是否合理（碩士論文通常5-6章）
- 章的順序是否符合「前言→抽象(上位概念)→具體→比較→檢討→結論與建議」
- 比較法章節是否獨立成章，且置於國內檢討之前
- 結論是否獨立成最後一章

**【新增】章名焦點檢查：章名是否與論文研究範圍相符？**

章名不得空泛地涵蓋整部法律或整個法律領域，應聚焦於論文真正要討論的特定問題或情境。判斷標準如下：

> 如果把章名獨立拿出來，讀者會預期讀到的內容，是否超出本論文實際要討論的範圍？

例如：論文題目是「生成式AI蒐集個資之規範研究」，第三章若命名為「個人資料保護法之現行規範」，讀者會預期看到整部個資法的介紹，但論文實際上只需要討論與生成式AI蒐集行為相關的條文。此時章名應改為「生成式AI蒐集個資之現行法律規範」，將範圍限縮至論文核心。

常見空泛章名的警示語：「○○法之規範」、「○○制度概述」、「○○法律架構」——出現這類字眼時，應主動詢問使用者：**這一章是要介紹整部法律，還是只討論與研究問題直接相關的部分？**

**等使用者確認第一層後，再進入第二層。**

### 第三步：調整第二層（節）

逐章列出各節標題，確認：
- 每章的節是否服務該章主題
- 跨章重複的內容是否合併
- 錯置的內容是否移回正確章節（例如：比較法內容不應散落在其他章）
- 每章是否有「小結」作為收尾

**【新增】章名與節次對應檢查**

章名中出現的每個核心概念，節次中必須有對應的節來承接，缺一不可。

> 例如：章名為「現行規範與草案架構」，節次中必須同時出現介紹現行規範的節與介紹草案架構的節，缺少任何一個都構成章名與節次脫節。

檢查方式：將章名拆解為各個核心概念，逐一比對節次是否有承接。若有概念未被任何節承接，應補入對應節次或修改章名。

**【新增】節名須標明討論對象**

同一章內若同時討論多個制度（如現行規範與草案、本國制度與外國制度），節名必須明確標示該節的討論對象，避免讀者混淆。

> 例如：同一章同時討論現行規範與草案，節名應標示為「草案之○○制度」或「現行規範之○○機制」，而非僅寫「○○制度」讓讀者自行判斷。

**【新增】制度介紹順序：全貌先於細節**

介紹任何制度時，應先設一節介紹該制度的整體架構與立法背景，再逐節展開具體機制，不得直接跳入細節。

> 例如：介紹草案時，應先有「草案之整體架構」節，再接「草案之許可制度」、「草案之資產保護機制」等具體節次。

### 第四步：確認章節間的對話關係

特別檢查：
- 第四章（外國制度）的結論，是否在第五章（國內檢討）中被引用？
- 第二章建立的理論原則，是否在第五章作為評價標準？
- 第六章的建議，是否有前面章節的論證支撐？

若看不出這些對話關係，需在第五章補充「比較法視角下之制度反思」節次作為橋樑。

**【新增】第五章開頭銜接節檢查**

第五章（國內制度檢討）的第一節，不得直接跳入實質建議（例如直接討論「歐盟模式之引進可行性」），必須先設置一個**過渡性前言節**，發揮以下三個功能：

1. 回顧第四章比較法研究的核心發現
2. 點明各外國模式對我國的啟示（不只單一國家）
3. 提出本章的檢討角度與評價標準，銜接後續修法建議

建議第一節標題格式為：「比較法研究之啟示與本章檢討方向」或「外國制度對我國修法之啟示」。

若第五章第一節直接進入單一國家制度的可行性討論，應提醒使用者：**第四章介紹了多個外國制度，第五章僅採用其中一個，其他國家的制度在此消失是否有意為之？若非刻意取捨，應在第一節統整所有比較法發現後，再進行篩選評估。**

## 常見問題與修正方向

| 問題 | 修正方向 |
|------|----------|
| 比較法內容夾在介紹制度的章節裡 | 獨立成第四章 |
| 上位概念（理論）與具體制度混在同一章 | 拆開為第二章（理論）與第三章（制度） |
| 實務案例放在結論之後 | 移至第五章作為制度缺陷的實證 |
| 修法建議散落在各章 | 集中至第五章末節或第六章 |
| 小結位置錯誤（放在其他節之前） | 小結永遠是該章最後一節 |
| 章節編號跳號（如第三節後直接第五節） | 檢查是否有遺漏節次 |
| **【新增】章名涵蓋範圍超出論文實際討論範圍（空泛章名）** | 將章名限縮至論文核心問題，例如「○○法之規範」→「生成式AI情境下之○○法規範」 |
| **【新增】第五章第一節直接進入單一國家制度評估，跳過比較法統整** | 在第五章首節加入過渡性前言節，統整所有比較法發現後再進行取捨評估 |
| **【新增】章名概念未被節次承接（章名與節次脫節）** | 將章名拆解為各核心概念，逐一補入對應節次，或修改章名使其與現有節次相符 |
| **【新增】同章討論多個制度但節名未標明討論對象** | 在節名加入制度名稱，如「草案之○○」、「現行規範之○○」，讓讀者一眼辨別 |
| **【新增】制度介紹直接跳入細節，缺乏整體架構節** | 在具體機制節之前補入「○○之整體架構」節，先給讀者全貌再展開細節 |
| **【新增9】章名出現文學性副標題（如賦劍篇、鑄盾篇、天平篇）** | 刪除文學性副標題，改以法律用語表達章節功能，例如「（賦劍篇：源頭打擊）」→直接刪除 |

---

## 注意事項

- **初步架構的章節**：若某章內容尚不確定（如外國法制研究不足），應在標題後加註「（初步架構，待補充）」，提醒後續補強。
- **碩士論文定位**：第二章的理論深度以「作為評價工具」為目標，不需要像博士論文一樣建立完整理論體系。
- **比較法的功能**：外國制度介紹的目的是「借鏡」，不是「移植」。第四章的小結應明確點出對我國的啟示，供第五章引用。

## 本次對談新增規則（v2）

### 格式規則

**【新增1】章名不得使用文學性副標題**

章名出現「賦劍篇」、「鑄盾篇」、「天平篇」等文學性副標題，應刪除，改以法律用語表達章節功能。

> 例如：「我國防堵避稅法制之檢討與重構（賦劍篇：源頭打擊）」→「我國防堵租稅規避法制之現狀與問題」

**【新增2】章名不得使用通用前綴**

章名不得以「理論基礎：」、「制度介紹：」等通用前綴開頭，應直接以實質內容命名。

> 例如：「理論基礎：租稅規避之憲政與法理界線」→「租稅規避之憲政與法理界線」

**【新增3】節名標點符號統一使用冒號**

節名中的標點符號應統一使用全形冒號（：），不使用破折號（——）。

> 例如：「租稅規避與稅捐逃漏之裁罰界線——以○○判決為中心」→「租稅規避與稅捐逃漏之裁罰界線：以○○判決為中心」

**【新增4】法條與法令名稱使用全名**

大綱中引用法條應使用全名。

> 例如：「納保法第7條」→「納稅者權利保護法第7條」；「北高行99年度訴字第543號」→「臺北高等行政法院99年度訴字第543號」

例外：法令名稱過長且已具辨識度者（如「所得稅法」、「稅捐稽徵法」）可使用簡稱，但應於第一次出現時標註全名。

---

### 結構規則

**【新增5】每章必須有前言節與小結節**

每章第一節應為「前言」，最後一節應為「小結」，缺一不可。前言點出本章核心命題，小結統整本章論點並銜接下一章。

**【新增6】上位概念的判斷標準**

判斷一個節是否屬於上位概念，標準如下：

> 該節的內容是否會在第五章被反覆引用作為評價標準？若只出現一次、或只服務單一章節，則不是上位概念，應移至對應章節。

常見錯置情形：
- 具體條文的法理與實務檢討（屬第三章）放入第二章
- 訴訟實務的核心理論（屬第五章）放入第二章

**【新增7】比較法只引用單一國家時須說明理由**

若比較法章節只介紹單一國家制度，應在緒論研究範圍節或比較法章前言，說明選擇該國的理由，預防口試委員質疑研究範圍過窄。

> 例如：只比較美國，應說明美國是全球避稅立法最完整的法域、所涉判決具比較法指標意義等。

---

### 論文題目規則

**【新增8】論文題目主從關係檢查**

論文題目應以「核心研究問題」為主標題，「研究手段或範圍限縮」為副標題。若題目讀起來像「我要研究X，順帶討論Y」，但實際上Y才是核心，應調整主從順序。

> 例如：「納稅者權利保護法修法方向之研究：以租稅規避為中心」（修法是手段，規避是核心）→「租稅規避防制與納稅者權利保護之衡平：以《納稅者權利保護法》第7條修正為中心」
